RESUMENSe explica qué es Git y por qué es una herramienta fundamental en el desarrollo de software. Se menciona que Git es un sistema de control de versiones que permite guardar el historial de cambios de un proyecto, lo cual ayuda a evitar la pérdida de información y facilita regresar a versiones anteriores en caso de que ocurra algún error. Además, se destaca que Git es un sistema distribuido, lo que significa que cada desarrollador posee una copia completa del repositorio en su propia computadora, permitiendo trabajar de manera más segura y eficiente. Posteriormente, el video muestra cómo instalar Git y cómo configurarlo por primera vez, estableciendo el nombre de usuario y el correo electrónico que se utilizarán para identificar los commits. Luego se enseña cómo crear un repositorio local utilizando el comando git init y se explica el funcionamiento de las tres áreas principales de trabajo en Git, el working directory, donde se realizan las modificaciones, el staging area, donde se organizan los cambios antes de confirmarlos, y finalmente el repositorio, donde se almacenan de manera permanente las versiones del proyecto.¿Qué es GIT?Es un sistema de control de versiones, como su nombre lo indica es un sistema, un programa que controla o administra las distintas versiones del proyecto. Fué creado por Linus Torvalds, el mismo creador del sistema operativo Linux. A medida que iba modificando el código, necesitaba una herramienta que le ayudara a administrar toda la complejidad de un proyecto de esa magnitud. Git permite tener un registro detallado de todos los cambios que se van realizando en el proyecto. Además facilita que múltiples personas desde diferentes partes del mundo puedan trabajar en el mismo código y seguir contribuyendo sin perder información ni generar conflictos.2. ¿Cuáles son los comandos más importantes y para que sirven?Git init: Es el comando que se utiliza para indicarle a Git que vamos a crear un nuevo proyecto. Inicializa un repositorio dentro de una carpeta y comienza a llevar el control de versiones. Git add <file>: Sirve para pasar los archivos del working directory al staging area. Es el paso previo entes de hacer un commit.Git status: Permite ver en qué estado se encuentran los archivos del proyecto, mostrando si están en el working directory, en el staging area o si ya fueron confirmados con un commit.Git commit: Sirve para pasar los archivos del staging area al repositorio local, creando un snapshot del estado actual del proyecto. Cada commit guarda una versión específica del código.Git push: Se utiliza para subir los cambios del repositorio local a un repositorio remoto, por ejemplo GitHub. Esto permite que otros desarrolladores puedan acceder al código y puedan trabajar en él.Git pull: Cuando se trabaja con múltiples desarrolladores, este comando permite traer a la computadora local los cambios que otros han realizado en el repositorio remoto.Git clone: Sirve para hacer una copia desde el servidor central donde está el código al computador para empezar a trabajar.3. ¿Cuál es la diferencia entre Git y Github?La diferencia principal es que Git es una herramienta o software que se instala en la computadora y permite controlar versiones de archivos de manera local. Es el sistema que registra los cambios y administra el historial del proyecto. Por otro lado, GitHub es una plataforma en línea que permite almacenar repositorios creados con Git en la nube. GitHub facilita compartir proyectos, hacer respaldo de la información y colaborar con otros desarrolladores. En resumen Git es la herramienta que controla las versiones y GitHub es el servicio que aloja esos proyectos en internet.4. Enumere los comandos para subir cambios a un repositorio.1. git init2. git add .3. git commit -m "mensaje"4. git remote add origin URL5. git push -u origin main5. ¿Qué es una branch?Es una línea de desarrollo independiente dentro de un mismo proyecto. Permite trabajar en nuevas funciones, mejoras o pruebas sin afectar la rama principal del proyecto, que normalmente se llama main. Las ramas son muy importantes cuando se trabaja en equipo, ya que cada desarrollador puede crear su propia branch para trabajar en una parte específica del proyecto y luego unirla a la rama principal cuando esté terminada. Esto mantiene el proyecto organizado y evita errores en la versión principal.
